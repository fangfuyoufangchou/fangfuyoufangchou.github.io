navigator的open-type属性值
发表于 2020-03-22| 更新于 2020-03-22
阅读量:|评论数:
open-type 描述
navigate 保留当前页面，跳转到应用内的某个页面。但是不能跳到 tabbar 页面，跳转到新页面左上角有返回键可以返回之前的界面
redirect 关闭当前页面，跳转到应用内的某个页面。但是不允许跳转到 tabbar 页面。跳转的界面左上角可以返回首页
switchTab 跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面
reLaunch 关闭所有页面，打开到应用内的某个页面,可以到tabBar页面
navigateBack 关闭当前页面，返回上一页面或多级页面和delta配套使用，delta=1表示返回1级,只有navigate跳转的页面才生效，可以返回tabBar页面
exit 退出小程序，target=”miniProgram”时生效，url不需要写，调试是没有效果的，只有真机上才有效果

注意：navigateBack只有由navigate跳转的页面才有效果，也就是可以返回，较为官方的说法是调用 navigateTo（navigate一样） 跳转时，调用该方法的页面会被加入堆栈,而navigateBack可通过 getCurrentPages获取当前的页面栈，决定需要返回几层。其它的方法则不会将页面加入堆栈，所以navigateBack在使用的时候就找不到页面，也就无法返回。
我个人觉得比较容易理解的解释是只有navigate进行页面跳转的时候不会关闭当前页面，其它在进行跳转的时候会关闭页面，navigateBack则是返回页面，你页面都关闭了，我怎么返回？返回到哪里？
当然如果你是先用navigate跳转，再在跳转页面使用redirect进行跳转，那么新的页面是可以返回上上一个页面，也就是naviagte跳转的页面，因为这个页面没有关闭。
不过现在小程序有默认的返回功能，也就是左上角，目前不太清楚navigateBack有什么特殊的用处，可能在自定义导航栏的时候会用到，本人目前也是刚刚开始学习，有理解错误的地方欢迎指出。
