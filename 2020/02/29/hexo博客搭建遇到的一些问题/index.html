hexo博客搭建遇到的一些问题
发表于 2020-02-29| 更新于 2020-03-01|杂谈
阅读量:|评论数:

今天终于把博客搭好了，感觉自己好菜，搭博客搭了半天，各种毛病不断出来。最奇怪的是，我明明是按照网上的教程来的呀，一步跟着一步，为啥还会出问题，百度了大半天，头都大了。索性最后还是成功了，虽然还有好多功能还没配置，但至少可以写东西了！！！可喜可贺。。。。。。个头呀。

算了，毕竟是第一次写博客，说实话，根本没有头绪。去看了下大佬们的博客，感觉好像有了点灵感，那就是瞎写！反正又没人看，我想怎么写就这么写！技术？那是啥？你有的话可以分点给我吗？

嗯嗯嗯嗯。。。废话好像说的有点多了，还是说正题。因为是第一次搭建（难道还有第二次？）问题肯定是遇到一大堆。

首先是安装，安装教程的话，网上一搜一大堆，我就不详细写出来了（其实是懒），下面贴一个我个人觉得比较好的安装教程-https://www.jianshu.com/p/8b704830797b

有可能出现的问题

  • 如果发现git操作不管是什么指令都报
    fatal: Not a git repository (or any of the parent directories): .git
    那应该是没有初始化本地仓库
    解决办法:输入git init
  • git commit提交报如下错误
    Code
    一般是没有git add到暂存区，需要先git add
  • 主题clone失败，报如下错误
    Code
    在网上查了有人说是文件只读，但我把只读取消，发现还是只读，改不了，后来又查了一下发现有说是GIT限制了推送数据的大小导致的
    解决方法如下：
    $ git config http.postBuffer 524288000
    如果报
    fatal: not in a git directory
    就加上–global试一下
    git config --global http.postBuffer 524288000
    如果两种都不行，那就不知道了。
  • commit之后 git status 查看状态，报如下代码
    Code
    说明本地有一个提交需要push到远程仓库，一般需要先push
    git push origin master

一般来说在用hexo搭建博客的过程中就遇到了这些问题，看起来少，但其实真正搭的话还是有一些其他问题，最重要的就是对git的使用不了解，网上找了一些答案也不知道在说啥，经常乱操作，结果把库搞乱了，又删除重新搭。浪费了好多时间。但好在现在还是完成了。对git的基本使用也有了写了解。下面的图可以方便记忆git库各个区域之间的关系，至于具体指令，网上一大堆，还是自己查去吧。我的图也是网上找的。
